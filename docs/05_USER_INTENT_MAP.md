# Harmonix V2

# User Intent Map

Version 2.0

---

# Purpose

This document defines the purpose of every major page.

Every page exists to help users accomplish exactly ONE primary objective.

A page should never attempt to solve multiple unrelated problems.

Whenever a new feature is proposed, first determine which user intent it supports.

If no page has that intent, reconsider whether the feature should exist.

---

Every page should follow the same hierarchy.

↓

↓

Level 4

Developer Information

---

# Universal Success Criteria

Regardless of page, users should always know:

Where am I?

What is happening?

What is my next action?

What is the current state of my assets?

Do I need to do anything?

If a page fails to answer these five questions, it should be redesigned.

---

# Final Principle

A page is successful when users complete their goal with confidence, not when every available piece of information is displayed.

Always optimize for decision quality over information quantity.
